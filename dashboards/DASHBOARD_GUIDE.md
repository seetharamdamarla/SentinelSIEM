# SentinelSIEM Dashboard Guide

This document describes all the key dashboards and saved queries used in the SentinelSIEM environment. These can be recreated in any Wazuh deployment using the steps below.

---

## Dashboard 1 — Security Events Overview

**Location:** Threat Hunting → Dashboard

**What it shows:**
- Total alert count over 24 hours
- Level 12+ (Critical) alert counter
- Authentication failure/success counters
- Top 10 alert groups evolution chart
- Top 5 rule groups pie chart
- Alert severity level distribution

**Key Filters to Apply:**
```
manager.name: ubuntu22
agent.id: <your_mac_agent_id>
Time: Last 24 hours
```

**Screenshot:** See `assets/01_dashboard_overview.png`

## Dashboard 2 — Custom Rule Alert View

**Location:** Threat Hunting → Events

**Purpose:** View all events generated by SentinelSIEM custom rules.

**Saved Queries:**

| Query Name | KQL Filter | Use Case |
|---|---|---|
| Custom FIM Alert | `rule.id: 100002` | View file tampering events |
| Brute Force Events | `rule.id: 5712` | View SSH brute force detected |
| Privilege Escalation | `rule.id: 5404` | View failed sudo attempts |
| All Custom Rules | `rule.id: (100001 OR 100002 OR 100003 OR 100004 OR 100005 OR 100006 OR 100007)` | View all custom detections |
| High Severity | `rule.level >= 10` | View HIGH and CRITICAL alerts only |

**How to save a query in Wazuh:**
1. Type the KQL filter in the search bar
2. Click **Save** (top menu)
3. Give it a name (e.g., "SentinelSIEM - Brute Force")
4. It will be available under **Open** next time

## Dashboard 6 — Raw Log Discovery (Threat Hunting)

**Location:** Discover (via hamburger menu → Explore → Discover)

**Index Pattern:** `wazuh-alerts-*`

**Key Fields to Add as Columns:**
- `timestamp`
- `agent.name`
- `rule.id`
- `rule.level`
- `rule.description`
- `rule.groups`
- `data.srcip`

**Powerful Threat Hunting Queries:**
```kql
# Find all authentication failures
rule.groups: authentication_failures

# Find high-severity events on macOS agent  
rule.level >= 10 AND agent.name: Seetharams*

# Find all FIM events
rule.groups: syscheck AND rule.groups: syscheck_entry_modified

# Find events from specific time window
@timestamp >= "2026-03-04T20:00:00" AND @timestamp <= "2026-03-04T23:00:00"
```

## 🔧 Importing/Recreating Dashboards

If you need to set up this environment from scratch:

1. **Install Wazuh** (Manager + Indexer + Dashboard) on Ubuntu 22.04
2. **Enroll macOS Agent** via `wazuh-authd` enrollment
3. **Deploy custom rules** from `rules/custom_detections.xml` to `/var/ossec/etc/rules/`
4. **Configure FIM** in macOS agent `ossec.conf` `<syscheck>` block with `realtime="yes"`
5. **Create CDB threat intel list** at `/var/ossec/etc/lists/malicious_ips`
6. **Run attack simulations** from `simulations/attack_guide.md`
7. **Navigate dashboards** as documented above to verify detections
